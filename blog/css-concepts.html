The Box Model October 18th 2015

Overview - The HTML Element Box Model

As we wrap up week 3 of Phase0 and look back at how much we've learned (command line, git, html, css) one of the most important topics that sticks out is method for defining the size and space taken by each element of html displayed on the page. Known collectively as the Box Model, the values of size for Content, Padding, Border and Margin define the visual shape of each element along with it's negative space in relation to bordering elements.

We'll discuss in more detail shortly but it's important to remember that two of the layers in the box model refer to positive space that will be visually represented on the page (Content and Border) while the other two layers are actually negative space between other layers (Padding and Margin). This is shown in the image below:


Content

The simplest of the layers to define, the content of every element on the page is simply a rectangular box. Whether defined explicitly by height and < width or implicitly by the substance of the content and allowing the browser's rendering engine to make decisions on the fly.

Padding

Padding then is the space outside the edge of the content but inside the border. As with the rest of the layers it can be defined as a length by value (pixels, em, etc ) or as a relative measures of the containing element (%) for each of the 4 sides independently (top, right, bottom, left).

Border

The border is a layer often used to highlight the 'edge' of a piece of content visually, though it is not the actual edge of the content defined by the box model and this can be confusing when working to lay out items from a wireframe. Borders can come in many different shapes and sizes but is defined by just 3 values: width, style and color. Each of these values can be set once for the entire border or controlled independently for each side, just like padding. Margin

Finally, margin is the layer on the extreme outer edge of the content and defines the negative space between the border of the element and where the next element's margin may start. Margin is defined the same way as padding, with a length for each side.

Conclusion

Though straightforward in defining, the box model is pretty powerful for controlling the look of elements on the page.
