Keeping Track - Arrays and Hashes October 25th 2015

Ruby provides two primary tools for storing collections of objects: Arrays and Hashes. In this post we'll explore both options and discuss briefly when you might want to use each

Arrays

Arrays are an ordered collection of objects (strings, integers or floats) where the index value (or key) for each object is an integer which represents its location within the list. Creating a new array is easy and can be done in a number of ways: 


                Empty Array:
                new_array = Array.new
                new_array = []
                or initialized with values:
                new_array = Array.new(3,2)
                new_array = [2, 2, 2]

Hashes

Hashes are similiar to Arrays in Ruby but have the added flexibility of using any String as the key for the ordered object. This ability to assign objects to a self-defined key allows us to create a dictionary of sorts. Hash key-value pairs are also locatable by Integer location within the index so in a sense Arrays are just a simplified subset of Hashes. Hashes are created in a similiar way as Arrays but use the brace instead of square bracket


                Empty Hash:
                new_hash = Hash.new
                new_hash = {}
                or initialized with values:
                new_hash = Hash.new("default value")
                new_hash = {"key","value","key","value"}

To Array or To Hash?

When evaluating how to collect a set of related information stored in Ruby objects the primary consideration betwen Hashes and Arrays are whether the information needs be classified with Keys that are not just a location. In plain english, whether the more intelligent classification of a piece of information like "123 Main Street" is in fact "Address" rather than "Index location 2".

One other consideration that will come into play later in our exploration of written good Ruby code is that Arrays execute more quickly than Hashes because they can be indexed more efficiently. More on this topic at a future date.
