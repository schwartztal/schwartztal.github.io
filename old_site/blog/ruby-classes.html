Ruby Classes

a neighborly example

Ruby classes are a useful way to define a set of characteristics and behaviors once for a set of related objects that can be used and re-used any number of times. The blueprint analogy for constructing a house has resonated most with me as we learn about Ruby classes: you create the framework by which all houses will be created and then quickly and efficiently create many houses which have the same behaviors but can still be unique through varying charcteristics. We'll stick with that analogy of a blueprint and examine how a Ruby class can help keep track of a growing Neighborhood.

Unlike in previous weeks where our work was focused entirely on an object or method and everything is done within that limited scope, when we enter the realm of defining classes we need to start considering the scope of variables we use. In this week we begin to use class and instance variables in addition to the local variable. Syntax of these new variables in Ruby is:

local variable - address (usable within the limited scope in which it is defined, typically a method or loop)
instance variable - @address (unique to an instantiated object of a class)
class variable - @@address (unique to that class and shared amongst all instances of the class, or "global within the class" )
global variable - $address (not used this week but relevant for context, this is accessible anywhere within Ruby)

Let's explore how to use class and instance variables with an example. Here we define a class "neighbors" that will create objects to hold information about each of my neighbors and a method to update family members.

class neighbors

@@neighbors = 0
@@neighborhood = "Cherry Hills"

attr_accessor :address
attr_accessor :family_members
attr_accessor :pets
attr_accessor :neighborhood

v def initialize (address, family_members, pets)
@address = address
@family_members = family_members
@pets = pets
@@neighbors += 1
end

def update_family(family_members)
@family_members = family_members
end

def update_neighborhood(new_neighborhood_name)
@@neighborhood = new_neighborhood_name
end
end

In this simple example we have a mix of variables scopes in use (local, instance and class). The local variable usage is limited to holding the arguments passed in during initalization of the object instance (i.e. "address"). Instance variables are used to then attach those arguments to the object and update the object when the update_family method is called with a new set of family_members. Finally the class variables @@neighbors and @@neighborhood are declared once when we set up the class and modified within the code.

The reason we use instance variables for address,family_members and pets is that the information these variables contain will be different for each instance of the class (each household) and so should be tied only to the object that is a part of that class.

The number of neighbors and name of the neighborhood are variables which are common to all households, and though they may change (new houses are built, or we vote to change the name) such a change would propogate immediately to all households and so it makes the most sense to keep them in the parent object of houseolds... which is the class neighbors
